Troubleshooting

Remember that you must have a Mac OS X driver for your device; Mac OS 9 drivers will not work. At the time of this writing, Mac OS X drivers are available for these devices:

  • MIDIMAN USB interfaces (See warning below!)
  • Mark of the Unicorn (MOTU) USB interfaces
  • Roland/Edirol interfaces

Mac OS X also supports any USB MIDI Class-compliant device, with no special driver necessary. This list is probably not complete.

  • Roland/Edirol UM-880
  • Event EZBus
  • Evolution keyboards (some older models may require a patch--see the web site)

Note: If you have a device hooked up, and the proper driver, but you don't see it in the menu, you may be running into a problem with the Classic environment. Classic tends to grab USB devices for its own exclusive use, and doesn't let programs on the native Mac OS X side get to them. Try shutting down Classic and starting SysEx Librarian again. (If SysEx Librarian is started first, and is running when Classic starts, you will not encounter this problem.)

Note: The MIDIMAN drivers are seriously brain-damaged and don't work well for sending SysEx. We try to work around the bugs, but things are still pretty unstable, and your machine may hang or SysEx Librarian may crash. Yell at MIDIMAN and tell them to fix their driver.

Note: The current MOTU driver (as of 29 December 2001) provides a source named "All Ports" for some devices. It has been reported that this source doesn't work--the interface's lights blink, but nothing is shown in MIDI Monitor. This is most likely a bug in the driver, and not MIDI Monitor's fault.
